Intentos de acceso agotados

El proceso sería el siguiente:

1. En la web se muestra un formulario para que el usuario introduzca un código.
2. El usuario introduce el código.
3. Se envía este código y lo recibe el servidor que es quién lo compara para ver si ya se ha usado o quedan intentos válidos.
4. Si el código no se ha usado las veces suficientes, el servidor responde y el guarda el numero de intentos usados.
5. Cuando sea el último intento válido, guarda la fecha y la hora en la que se ha usado.
6. Si no quedan intentos disponibles, el serevidor responde con que se han agotado los intentos y guarda el número de intento no válido,para mostrarlo en la siguiente consulta.
7. El servidor es el que responde con la información guardada.
8. El cliente muestra la información que tiene guardada el servidor.

¿Qué hace el cliente?

Muestra la interfaz de usuario, que permite al usuario interactuar con la aplicación

¿Qué hace el servidor?

Guarda la información de los intentos y responde con la información que tiene guardada.
